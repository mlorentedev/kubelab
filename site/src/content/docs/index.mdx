## Why KubeLab?

Managed Kubernetes teaches you how to click buttons. It doesn't teach you how platforms work.

KubeLab is a production-grade IDP built from bare metal. Five physical nodes, three Proxmox VMs running K3s, a Raspberry Pi as network gateway, and a Jetson Nano for ML inference. Every layer — hardware, networking, orchestration, auth, observability — is configured by hand and version-controlled.

## Stack

<CardGrid>
  <Card title="K3s on Proxmox" icon="rocket">
    3-node cluster on mini PCs. Kustomize overlays for staging and production. Automated provisioning with Ansible.
  </Card>
  <Card title="Headscale VPN Mesh" icon="network">
    Self-hosted Tailscale coordination server. 9 nodes across VPS, home lab, and remote machines. Split DNS via Pi-hole + CoreDNS.
  </Card>
  <Card title="Authelia SSO" icon="approve-check">
    Single sign-on with OIDC for every service — Grafana, MinIO, n8n. Forward-auth on Traefik IngressRoutes. Two-factor enforcement.
  </Card>
  <Card title="Full IaC" icon="setting">
    Terraform for Cloudflare DNS. Ansible for node provisioning. Kustomize for K8s manifests. SOPS for secrets. Nothing is manual.
  </Card>
</CardGrid>

## Hardware

```
VPS Hetzner ── Production (Docker Compose, migrating to K3s)
Acemagic x2  ── K3s server + agents (Proxmox, 12GB each)
Beelink      ── Ollama inference (8GB, bare metal)
RPi 4        ── Network gateway: Pi-hole, CoreDNS, Headscale relay
RPi 3        ── External monitoring (Uptime Kuma)
Jetson Nano  ── GPU inference (llama.cpp)
```

## Services

Traefik ingress, Grafana + Loki + Prometheus observability, CrowdSec WAF, Gitea, n8n automation, MinIO object storage — all behind Authelia, all on Kubernetes.
