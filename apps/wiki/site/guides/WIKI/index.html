    🎯 Escenarios de Uso Comunes¶

    Scenario 1: "Nuevo desarrollador se une al equipo"¶

    Ruta de lectura: ⏱️ ~30 minutos

    1. ARCHITECTURE - Entender el proyecto (5 min)
    2. ADR - Por qué tomamos estas decisiones (10 min)
    3. CONTRIBUTING - Cómo trabajamos (10 min)
    4. HOW-TO - Comandos básicos (5 min)

    Setup práctico:

    git clone <repo>
make install-precommit-hooks
make up
# Leer HOW-TO para comandos diarios

    Scenario 2: "Problema en producción, debug urgente"¶

    Ruta de diagnóstico: ⏱️ ~5 minutos

    1. HOW-TO: Debugging - Comandos básicos
    2. TROUBLESHOOTING - Problemas específicos
    3. Si necesario: DEPLOYMENT - Config servidores

    Comandos inmediatos:

    make status ENV=production
make logs ENV=production
curl -f https://mlorente.dev/health

    Scenario 3: "Quiero añadir nueva funcionalidad"¶

    Proceso completo: ⏱️ ~45 minutos

    1. CONTRIBUTING: Pull Request - Proceso (10 min)
    2. CONTRIBUTING: Convenciones - Código (15 min)
    3. HOW-TO: CI/CD - Testing (10 min)
    4. CI-CD: Workflows - Si toca CI/CD (10 min)

    Scenario 4: "Setup servidor nuevo"¶

    Proceso completo: ⏱️ ~2 horas

    1. DEPLOYMENT: Requisitos - Hardware/OS (10 min)
    2. DEPLOYMENT: Setup Inicial - Usuario/SSH (20 min)
    3. DEPLOYMENT: Configuración - Env específico (30 min)
    4. HOW-TO: Deploy - Primer deploy (20 min)
    5. DEPLOYMENT: Monitoring - Setup monitoring (20 min)
